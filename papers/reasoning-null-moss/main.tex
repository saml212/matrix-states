\section{Registered Gates and Routing Rules}
\label{app:gates}

This appendix reproduces, in compact form, the pre-registered gates and
mechanical routing rules the main text relies on. Each was fixed in the
program's design registry before the corresponding data existed.

\textbf{Geometric-readout validity gates (waves 1--4).} A cell's readout is
valid only if all of the following hold. (a) \emph{$h{=}1$ sanity floor:}
the one-hop recovered fraction must exceed the upper edge of a
shuffled-pairing null by more than the null's width, and must also exceed
an absolute floor of 0.10. (b) \emph{Premise (iii):} the median cosine
between an entity's query projection and the same entity's key projection
must exceed the 95th percentile of a shuffled-entity null. (c)
\emph{Premise (iv):} the median cosine between the key and value
projections of the same token must exceed the same null percentile.
\emph{Routing:} failure of any gate routes the cell to probe-invalid; the
registered rule states that gate failure never routes to refutation.
The readout's recovery math (state-power application and the
cosine-recovery scorer) is unit-tested against hand-built matrices.

\textbf{The positive control (2026-07-11): failed, root-caused, fixed,
independently reviewed.} A pre-submission positive control was run
end-to-end through a real forward pass on the production multi-layer
kernel path, closing the gap the previous paragraph flagged as open. A
synthetic $K{=}32$ mutually-orthonormal-key single cycle was injected at
the readout layer of a real trained checkpoint (mutually orthonormal keys
make the correct $h$-hop answer exactly the $(a{+}h) \bmod K$ key, so
recovery is analytically known), scored through the identical
production \texttt{squeeze\_state\_head} $\to$ \texttt{apply\_state\_power}
$\to$ cosine-recovery code path every grid cell uses.
% evidence: C10-instrumentfix
\emph{Result.} The production path recovered 0 of 256 at every $h$
(cosine indistinguishable from noise), while a manually-transposed variant
of the identical state recovered 1.0000 at every $h$ through the same
scoring code; diagnostics ruled out a script artifact (key orthonormality
to $3.6{\times}10^{-7}$, zero hook-injection deviation, no near-zero
$\beta$ items, and the real kernel's raw state matches an independently
implemented reference recurrence to 0.17\% relative Frobenius error).
% evidence: C10-instrumentfix
\emph{Root cause.} A single-write closed-form adjudication (one basis-vector
key/value pair, all other positions state-neutral, computed independently
of any probe code) confirmed \texttt{squeeze\_state\_head} returned the
underlying kernel's raw $[K,V]$ (key-major) state layout unchanged, while
\texttt{apply\_state\_power} contracts its query vector against what it
assumes is the last (key) axis of a $[V,K]$ (value-major) layout: the
readout output matched the $[K,V]$ closed form at 0.0 relative Frobenius
error and the $[V,K]$ closed form at exactly $\sqrt{2}$ (the pure-transpose
signature), and the two closed forms were themselves exact transposes of
each other. This is the same defect class as a previously corrected
defect elsewhere in this program's codebase, never inherited here because
this readout implements an independently written state-extraction function.
% evidence: C10-instrumentfix
\emph{Fix and review.} The fix is a one-line transpose,
\texttt{final\_state[:,\,0,\,:,\,:].transpose(-1,-2)} in place of the
untransposed slice, mirroring the already-corrected reference function's
own transpose exactly. A hand-computed regression test asserts both the
post-fix recovery and, as a kill-proof, that the pre-fix layout inlined
still fails the identical assertion. An independent review (a fresh
reviewer with no access to the fixing session's own reasoning) confirmed
the diff is minimal and correctly scoped: \texttt{squeeze\_state\_head} has
exactly one production call site feeding \texttt{apply\_state\_power} (the
intended target) and the state condition-number covariate, which is
provably transpose-invariant ($\mathrm{svdvals}(S) = \mathrm{svdvals}(S^\top)$),
so no other production code path changes behavior. Re-running the positive
control under the fixed function recovers 1.0000 at every $h$
(cosine $\approx 0.99999$), and the manually-transposed arm, now playing
the pre-fix-equivalent wrong-layout role, recovers 0.0000: the exact
role-swap the closed-form adjudication predicts.
% evidence: C10-instrumentfix

\textbf{The re-metric (78/320 nonzero) and why it does not reopen the
lane.} Re-running the full wave-1/wave-2 grid -- 80 cells, 320 (cell,$h$)
readings, identical archived checkpoints and deterministic episode seeds,
one forward pass per cell -- under the fixed function does not reproduce
the pre-fix zero: 78 readings are nonzero (30 of 60 intervention-grid
cells, 8 of 20 scale-ladder cells -- the re-metric's own Leg-A/Leg-B
partition, a different axis from Table~\ref{tab:program}'s wave-1 (78
cells) / wave-2 (2 cells) split of the same 80 cells -- carry $\geq 1$
nonzero hop), $h{=}1$ recovered fraction
reaching 0.8691 at its strongest cell (a per-token-arm wikitext cell,
seed 1, $K{=}32$) and $h{\geq}2$ reaching 0.6375 in one cell. The
alignment-premise gates read key/query/value hook values, never the
state-extraction output the defect touched, and continue to fail at all
320 readings (0/320 both premises) before and after the fix, so under the
registered routing rule this reading alone licenses no confirm regardless
of what follows. % evidence: C11-remetric

Two correspondence nulls, pre-registered before either ran, were used to
test whether the fixed-lens signal reflects genuine state--target
correspondence or is trivially satisfiable by any (state, wrong-target)
pairing. The first, a label-shuffle null (the readout's existing
null-relative machinery, extended from one previously-exercised
calibration cell to all 80 grid cells), grades the identical forward
pass's prediction against an independently drawn Hamiltonian $K$-cycle
rather than the true one. This null reproduces the real reading at every
one of the 320 readings (0/320 clear the pre-registered bar: null $<$ half
the real value, AND real exceeds the null's bootstrap upper edge by the
null's own width, AND real $\geq 0.10$); at $h{=}1$ specifically, real and
null read identically at every strong cell (e.g.\ 0.8691/0.8691), traced to
a construction property rather than a grading bug: the $h{=}1$ target is
computed with zero permutation iterations, so it never consults the
shuffled cycle at all, making the label-shuffle null vacuous at $h{=}1$ by
construction in every cell, including the readout's own original
calibration cell (a healthy-draw check separately confirmed the null cycle
itself agrees with the true one on only 4.1\% of entries, ruling out a
degenerate shuffle). A second, positional derangement null was therefore
pre-registered as the anticipated fallback: it grades the same prediction
against a fixed-point-free random permutation of the candidate value
slots, breaking correspondence at every hop including $h{=}1$. This null
also reproduces the fixed-lens signal at every one of the 320 readings
(0/320 clear; the strongest cell, 0.8691 real, reads 0.8125 against the
deranged target, 94\% reproduction against the pre-registered 50\% bar),
and was itself kill-proofed with a perfect-correspondence fixture
(real 1.0, deranged 0.0) and a collapsed-geometry fixture
(real $\approx$ deranged) before being trusted.
% evidence: C12-nullvalidation

The mechanism both nulls point to is geometric, not compositional: at the
readout's strongest cells the cross-checkpoint prediction direction
converges (mean pairwise cosine 0.9996 across queries, under state
condition numbers of $1.8{\times}10^4$ to $2.1{\times}10^6$) while the
candidate value population is itself near-collinear (mean pairwise cosine
0.9648); a near-constant prediction direction scores cosine above 0.9
against nearly any member of a near-collinear population, real or
deranged target alike. \textbf{Provenance flag: these two covariates
(0.9996, 0.9648), and the 4.1-percent healthy-draw check above, are a
design-registry record of an on-box diagnostic on the single strongest
cell, not backed by a separately archived raw artifact.} No raw state
tensors were saved by the original harvest to recompute them from, so the
empirical verdict above (0/320 clear either null) does not depend on them,
but this specific mechanistic interpretation of \emph{why} does, and is
reported with that provenance disclosed rather than independently
re-derived.
% evidence: C12-nullvalidation
An independent episode-resampling check (a fresh deterministic episode
draw on the identical checkpoints) confirms the underlying bimodality --
which cells carry any signal at all -- is a stable property of the trained
state (20 of 21 floor-clearing cells stay on under a resampled episode),
not an artifact of one lucky draw; within this family, the deployed
per-token intervention arm concentrates the effect (20 of 72 $h{\geq}2$
readings clear a 0.10 floor, versus 1 of 36 for the off arm and 0 of 72
for the global arm): concentration of the artifact, not of composition,
per the same two nulls. % evidence: C12-nullvalidation

\textbf{The corrected claim, binding throughout this paper.} Geometric
recovery is statistically indistinguishable from a broken correspondence at
every scale, corpus, and hop the corrected instrument was run on. It is
not that recovery reads exactly zero, which was true only of the pre-fix
instrument on this sub-grid, and it is not a confirmed correspondence,
which both correspondence nulls exclude. Waves 3 and 4 (natural-language
and task-familiarized deployments, Section~\ref{sec:geometric}) call the
identical pre-fix state-extraction function and were not independently
re-run under the fix or tested against either correspondence null; their
probe-invalid routing rests entirely on the alignment-premise gates, which
the defect never touched and are therefore unaffected, but their own raw
zero readings carry the same unconfirmed status the wave-1/wave-2 zero
readings carried before this re-verification. That status is reported
here as disclosed, not assumed corrected by extension.
% evidence: C11-remetric, C12-nullvalidation

\textbf{Behavioral-contrast classification (wave 5).} The per-checkpoint
arm effect is $\Delta(c) = L_q(\text{control}, c) - L_q(\text{arm}, c)$,
with three-seed 95-percent intervals ($t(2, .975) = 4.303$). % evidence: C5-power
A checkpoint
satisfies the differential condition when the $K{=}32$ interval excludes
zero, the $K{=}20$ interval does not, and $|\Delta_{32}| > |\Delta_{20}|$.
Trajectories classify into six registered outcomes: persistent, transient
(the condition holds at an interior checkpoint and fails at the terminal
one), late-emergent, converged-equivalent, unresolved, and a
familiarization-null exclusion. The three-seed power floor (a detectable
effect of roughly 1.5 to 1.7 loss units, derived from the control cells'
between-seed spread) was registered together with the instruction that
unresolved outcomes be reported as measurement bounds. % evidence: C5-power

The registered pipeline computed one classification per corpus, using the
global arm as that corpus's representative signal, and classified both
corpora unresolved. % evidence: C5-power
The four-way (corpus, arm) breakdown reported in the
main text and Table~\ref{tab:program} applies the same registered
classification primitives to each arm's own trajectory. This finer split
was a build-time re-derivation performed after the corpus-level output was
seen; it was later folded into the analysis code and re-validated against
the archived per-cell deltas, reproducing the same four verdicts. It is
disclosed here as a re-derivation, not presented as the registered
corpus-level verdict.

The primary table computes 40 uncorrected 95-percent intervals (4
contrasts $\times$ 5 checkpoints $\times$ 2 loads); five exclude zero:
both loads of reasoning-dense$\times$global at checkpoint 1{,}000
($K{=}32$: $\Delta{=}{-}0.203$, $[-0.402,-0.004]$; $K{=}20$:
$\Delta{=}{-}0.199$, $[-0.325,-0.074]$), both loads of
encyclopedic$\times$per-token at checkpoint 1{,}000 ($K{=}32$:
$\Delta{=}{-}0.168$, $[-0.301,-0.036]$; $K{=}20$: $\Delta{=}{-}0.109$,
$[-0.211,-0.006]$), and the $K{=}32$ interval of
encyclopedic$\times$per-token at checkpoint 2{,}500 ($\Delta{=}{-}0.500$,
$[-0.624,-0.376]$, the reported transient). % evidence: C6-transient
Only the last also satisfies
the compound differential condition above and is corroborated by the
independent held-out-hop readout at the same cell and checkpoint; that
compound condition, not any single interval exclusion, is what the main
text calls the one determinate signal. The checkpoint-1{,}000 cells fail
the differential condition precisely because their $K{=}20$ intervals
also exclude zero.

\textbf{Replication integrity gates (wave 6).} Archived seeds load from
stored values only; a runtime guard raises on any live re-scoring of an
archived seed. Before pooling, a batch-effect gate requires (a) cohort
mean shift below twice the pooled standard error (Welch form) and (b)
cohort variance ratio below a pre-registered heuristic 4.0 (not a
calibrated $F$-test; the one-sided 5-percent $F(2,8)$ critical value for
the archived three-seed versus new nine-seed cohorts is $\approx 4.46$).
% evidence: C7-replication
On gate failure the registered
routing reports both cohorts separately, produces no decision-grade pooled
interval, and classifies the wave as batch-effect-flagged, outside the
four-outcome partition (confirmed at magnitude, confirmed smaller,
refuted, new pattern) that a passing gate would have entered.

\section{Reproducibility}
\label{app:repro}

Every number in this paper is computed from archived raw result files
(per-cell JSON records written by the experiment chains at run time), not
from summaries. All figures regenerate from one versioned script that
loads only those archived files and asserts the md5 checksum of every file
it reads against a recorded manifest, so a changed or substituted artifact
fails the build rather than silently altering a panel. As a compute
disclosure for this venue's small-scale remit: all experiments ran on one
or two GPUs at a time, and the six waves consumed approximately 9.5
GPU-hours in total. % evidence: C9-compute
The instrument re-verification (Section~\ref{sec:geometric},
Appendix~\ref{app:gates}) -- the positive control, the closed-form
adjudication, the 80-cell re-metric, and both correspondence-null sweeps
-- consumed a further $\approx$0.52 GPU-hours on one GPU, disclosed
separately since it postdates the original six-wave harvest.
% evidence: C10-instrumentfix, C11-remetric, C12-nullvalidation
The archived raw files, the figure script with its checksum manifest, the
experiment chains, and the full pre-registration text will be released
with the camera-ready version.
